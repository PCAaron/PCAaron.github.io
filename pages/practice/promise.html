# promise

# 什么是promise

Promise 是异步编程的一种解决方案：
从语法上讲，promise是一个对象，从它可以获取异步操作的消息；
从本意上讲，它是承诺，承诺它过一段时间会给你一个结果。

# promise 规范

promise规范是一个开放的，可实现的，可互操作的JavaScript承诺的标准。

promise标准中规范了主要两点：

  • 一个promise的当前状态只能是pending、fulfilled和rejected三种之一。状态改变只能是pending到fulfilled或者pending到rejected。状态改变不可逆。
  • promise的then方法接收两个可选参数，表示该promise状态改变时的回调(promise.then(onFulfilled, onRejected))。then方法返回一个promise，then 方法可以被同一个 promise 调用多次。

其中，Promise/A+中并没规范race、all、catch方法，这些是ES6制定的。

# promise解决的痛点

# 引入promise的原因及用于解决问题

在实际的应用中，其中有一个主要的场景就是异步请求，由于请求的过程是异步的， 需要时间等待回调结果才能继续进行。在这种场景下，如果存在多个ajax异步的请求 并且事件回调存在依赖的请求下，就会导致地狱回调的问题。这样，一方面 会致使代码的可读性较低；另一方面会导致可维护性降低。

通过promise的使用，可以很好的避免地狱回调的发生。通过promise的链式调用可以 增强代码的可读性，并且由于promise的状态不可逆，所以能避免地狱回调的场景和 处理回调函数的后续执行。

# promise的使用方式

  • 链式调用
const promise = new Promise((resolve,reject)=>{
    if(2>1){
        resolve('success')
    }else{
        reject('fail')
    }
})
promise.then(val=>{
    console.log(val)
},reason=>{
    console.err(reason)
})      
// success
  • Catch 的后续链式操作
new Promise((resolve, reject) => {
    console.log('初始化');

    resolve();
})
.then(() => {
    throw new Error('有哪里不对了');
        
    console.log('执行「这个」”');
})
.catch(() => {
    console.log('执行「那个」');
})
.then(() => {
    console.log('执行「这个」，无论前面发生了什么');
});

/*
初始化
执行“那个”
执行“这个”，无论前面发生了什么
*/
  • Promise.all()
// promise.all
let p1 = new  KPromise((resolve,reject)=>{
    resolve(1)
})
let p2= new KPromise((resolve,reject)=>{
    setTimeout(()=>{
       resolve(3) 
    },1000)
})
KPromise.all([p1,p2]).then(data=>{
    console.log(data)
})
  • Promise.race()
const promise1 = new Promise(function(resolve, reject) {
    setTimeout(resolve, 500, 'one');
});

const promise2 = new Promise(function(resolve, reject) {
    setTimeout(resolve, 100, 'two');
});

Promise.race([promise1, promise2]).then(function(value) {
  console.log(value);
  // Both resolve, but promise2 is faster
});
// expected output: "two"

                    })

            setTimeout(()=>{
                if(this.state = PENDING){
                    this.state =REJECTED
